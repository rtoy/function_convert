1.1 Overview

The function_convert package rewrites all occurrences of one function in terms of another within a Maxima expression. Specifically, a converter is a rule f = g that specifies how calls to f should expressed using the function g. Converters may express identities, special-function relations, or any user-defined transformation that can be written as a Maxima function.

Unlike literal renaming, a converter expresses a semantic relationship. For example, the converter sinc = sin does not literally replace the symbol sinc by sin; instead it applies the identity sinc(x) = sin(x)/x, replacing the left-hand side with the right-hand side in a given Maxima expression. Converters may be chained, and the system maintains an internal directed graph of all available conversions.

Specifically, a rule has the form f = g, where f is the source function and the target function g is either:

  • - the name of a target function,
  • - a Maxima lambda expression, or
  • - the name of a Maxima function, possibly user-defined.

Specific and General Conversion Rules

A rule of the form f = g, where f is the name of a Maxima function, applies only to calls of the specific function f. For example, 

(%i1) function_convert(sin = exp, sin(x));
(%o1) -(((%e^(%i*x)-%e^-(%i*x))*%i)/2)

converts sin(x) to its exponential form, while 

(%i2) function_convert(sin = exp, cos(x));
(%o2) cos(x)

leaves cos(x) unchanged. No other functions are affected by a rule that names a single function.

More general rules, such as trig = exp, apply to all functions in the corresponding family. Thus 

(%i3) function_convert(trig = exp, cos(x) + sin(x));
(%o3) (%e^(%i*x)+%e^-(%i*x))/2-((%e^(%i*x)-%e^-(%i*x))*%i)/2

converts all trigonometric functions to their exponential representation.

Available Conversion Classes

As we have seen, some conversion rules apply not only to a single function but to an entire family of related functions. The following class names may be used on the left-hand side of a rule such as trig = exp.

trig

The six trigonometric functions: sin, cos, tan, sec, csc, cot.

hyperbolic

The six hyperbolic functions: sinh, cosh, tanh, sech, csch, coth.

inv_trig

The six inverse trigonometric functions: asin, acos, atan, asec, acsc, acot.

inv_hyperbolic

The six inverse hyperbolic functions: asinh, acosh, atanh, asech, acsch, acoth.

exp

The exponential operator exp (including expressions of the form a^b).

gamma_like

Gamma-related special functions: gamma, beta, binomial, double_factorial, factorial, and pochhammer.

bessel

The Bessel family: bessel_j, bessel_y, bessel_i, bessel_k, hankel_1, hankel_2.

algebraic

Basic algebraic operators: +, *, and exponentiation.

inequation

Relational operators: =, <, <=, /=, >, >=.

logarithmic

The natural logarithm log.

Multiple converters may be applied in sequence. When the user supplies a list of converters, they are applied from left to right. When the system searches for a path automatically (via find_conversion_path), it uses a breadth-first search of the converter graph to find a shortest path.

The converter system does not modify Maxima’s simplifier; all conversions occur only when explicitly requested by the user.

The user-level interface consists of functions for applying converters, inspecting the converter graph, constructing conversion paths, and registering or deleting user-defined converters.

1.2 User-Level Functions

The following functions make up the user-level interface to the converter system:

  • function_convert(converters, expr) Apply one or more conversion rules to an expression. See function_convert.
  • list_converters([name_1, name_2, ...]) List all registered converters, optionally filtered by source function. See list_converters.
  • describe_converter(f = g) Display detailed information about a specific converter. See describe_converter.
  • delete_converter(rules) Delete one or more user-defined converters. See delete_converter.
  • delete_all_user_converters() Remove all user-defined converters at once. See delete_all_user_converters.
  • find_conversion_path(from, to) Construct a list of adjacent conversion steps forming a path from from to to. See find_conversion_path.
  • register_converter(spec, fn, doc) Register a new converter, optionally with an alias and documentation. See register_converter.

1.3 function_convert

Function: function_convert (rules, expr) ¶

Apply one or more function-conversion rules to the expression expr. The argument rules is either a single conversion rule f = g or a Maxima list of such rules.

A conversion rule has the form f = g, where f is the source function and g is either the name of the target function, a Maxima lambda expression, or the name of a user-defined Maxima function.

The operator = indicates a semantic conversion, not a literal renaming. For example, the rule sinc = sin does not replace the symbol sinc by sin. Instead, it applies the identity sinc(x) = sin(x)/x. Specifically: 

(%i1) function_convert('sinc = 'sin, sinc(x) + 1);
                                  sin(x)
(%o1)                             ------ + 1
                                    x

Users may also supply an explicit conversion by giving a lambda expression on the right-hand side. For example: 

(%i2) function_convert('erfc = lambda([q], 1 - erf(q)), erfc(x)^2);
                                             2
(%o2)                            (1 - erf(x))

Multiple conversions may be given in a Maxima list; they are applied from left to right.

Example 

(%i3) function_convert(['sinc = 'sin, 'sin = 'exp], sinc(x));
                                %i x     - %i x
                             (%e     - %e      ) %i
(%o3)                      - ----------------------- 
                                      2 x

1.4 list_converters

Function: list_converters ([name_1, name_2, ...]) ¶

Return a list of all registered function converters known to function_convert. In addition to returning this list, each converter is printed as a pair FROM = TO followed by a short description of the converter.

When called with no arguments, list_converters returns the full set of converters. When one or more names are supplied, only those converters whose source function matches one of the given names are returned. 

(%i2) list_converters();
psi = psi_half :  Apply the polygamma half-argument identity; see DLMF
     http://dlmf.nist.gov/5.5.E8

inequation = zero_lhs :  Normalize an inequation `a op b` to the zero LHS form
     `(a - b) op 0`.

...

The list of converters is maintained internally and entries appear in an unspecified order.

1.5 describe_converter

Function: describe_converter (rule) ¶

Describe the converter specified by rule, which must be a single converter of the form f = g. If a converter from f to g is defined, describe_converter prints the converter, indicates whether it is built-in or user-defined, and displays a short description if one is available. The function returns the symbol done.

If no such converter exists, a message stating this is printed. If the input is not a valid converter equation, an error is signaled. 

(%i1) describe_converter(sinc = sin);
Converter: sinc = sin
Type: built-in
Docstring: Convert sinc(x) into sin(x)/x.
(%o1)                                done

(%i2) describe_converter(a = b);
No converter defined for a = b
(%o2)                                done

(%i3) describe_converter(a < b);
Bad transformation (missing =): a < b

1.6 delete_converter

Function: delete_converter (rules) ¶

Delete one or more user-defined converters. The argument rules may be either a single converter rule of the form f = g or a list of such rules. The function returns the symbol done.

If a listed converter is built-in, an informational message is printed and the converter is not deleted; for example: 

(%i1) delete_converter(sinc = sin);
Cannot delete built-in converter sinc = sin.
(%o1)                                done

If a listed converter does not exist as a user-defined converter, a message is printed and nothing is removed: 

(%i1) delete_converter(a = b);
No converter a = b exists.
(%o1)                                done

The following example illustrates deletion of a user-defined rule. The rule applies radcan to every subexpression of the form a^b, but does not apply radcan to the entire input. After deleting the rule, applying it returns the input unchanged: 

(%i1) register_converter("^" = radcan, lambda([a,b], radcan(a^b)));
(%o1)                                done

(%i2) function_convert("^" = radcan, (1+x)^2 + a*(b+1));
                            2
(%o2)                      x  + 2 x + a (b + 1) + 1

(%i3) delete_converter("^" = radcan);
Deleted converter ^ = radcan.
(%o3)                                done

(%i4) function_convert("^" = radcan, (1+x)^2 + a*(b+1));
                                    2
(%o4)                        (x + 1)  + a (b + 1)

1.7 delete_all_user_converters

Function: delete_all_user_converters () ¶

Delete all user-defined converters. All built-in converters are preserved. A message is printed indicating that the user-defined converters have been removed. The function returns done. 

(%i1) register_converter("^" = radcan, lambda([a,b], radcan(a^b)));
(%o1)

(%i2) delete_all_user_converters();
All user-defined converters deleted.
(%o2)                                done

1.8 find_conversion_path

Function: find_conversion_path (from, to) ¶

Find a shortest conversion path from from to to. The function performs a breadth-first search of the full converter graph and returns one such path, represented as a Maxima list of converter rules f = g. If several shortest paths exist, one is chosen according to the system’s internal graph-traversal order; the choice among equal-length paths is not user-configurable. All properly registered user-level converters may appear as links in a path.

The function find_conversion_path does not perform any conversions. To apply the path, call function_convert with the list returned by find_conversion_path. If no path exists, the empty list [] is returned. 

(%i1) path : find_conversion_path(sin, gamma);
(%o1) [sin = sinc, sinc = gamma]

(%i2) function_convert(path, sin(x));
(%o2) x/(gamma(1 - x/%pi) * gamma(x/%pi + 1))

1.9 register_converter

Function: register_converter (rule, fn, &optional doc) ¶

Register a new converter.

The argument rule is either a single converter rule of the form f = g, or a two-element Maxima list [f = g, p = q]. In the first case, the primary converter is f = g and it has no alias. In the second case, the first element of the list is primary converter rule and the second element is its alias.

If either the primary rule or the alias rule is not of the correct form, an error is signaled. If a converter from f to g already exists, an error is also signaled: built-in converters cannot be redefined, and redefining a user-defined converter is allowed, but a informational message is printed.

The argument fn must be either a Maxima function or a Maxima lambda form; otherwise an error is signaled. If a string doc is supplied, it is stored as the documentation for this converter, and is displayed by describe_converter.

The function returns the symbol done.

Here we define a converter that applies radcan to all subexpressions of the form a^b. Since the source function "^" is a function of two variables, the associated function must be a function of two variables. Notice that in (%i2), the term a*(b+1) is not touched by radcan. 

(%i1) register_converter("^" = radcan, lambda([a,b], radcan(a^b)));
(%o1)                                done

(%i2) function_convert("^" = radcan, (1+x)^2 + a*(b+1));
                            2
(%o2)                      x  + 2 x + a (b + 1) + 1

Instead of using a Maxima lambda form for the function, we can use the name of a Maxima function; for example 

(%i1) fff(a,b) := radcan(a^b)$

(%i2) register_converter("^" = radcan, fff);
(%o2)                                done
(%i3) function_convert("^" = radcan, (1+x)^2 + a*(b+1));
                            2
(%o3)                      x  + 2 x + a (b + 1) + 1

An example of including an alias: 

(%i3) register_converter(["^^" = expand, ppp = qqq], lambda([a,b], expand(a^^b)));
(%o3)                                done

(%i4) function_convert("^^" = expand, (x+1)^^2);
                                 <2>
(%o4)                           x    + 2 x + 1

(%i5) function_convert(ppp = qqq, (x+1)^^2);
                                 <2>
(%o5)                           x    + 2 x + 1

Redefining a built-in converter signals an error: 

(%i1) sinc_to_sin(x) := sin(x)/x$

(%i2) register_converter(sinc = sin, sinc_to_sin);
Cannot redefine built-in converter: sinc = sin

But redefining a non-built-in converter is allowed: 

(%i3) register_converter(sinc = fff, sinc_to_sin);
(%o3)                                done

(%i4) register_converter(sinc = fff, sinc_to_sin);
Redefining converter sinc = fff
(%o4)                                done

1.10 Built-in Converters

The function list_converters() displays all registered converters, including user-defined ones, together with a short description when available. The built-in converters are:

psi = psi_half

Apply the polygamma half-argument identity. See DLMF http://dlmf.nist.gov/5.5.E8.

bessel = bessel_recursion

Rewrite Bessel functions using standard recursion identities.

inequation = zero_lhs

Normalize an inequation a op b to the zero-LHS form (a - b) op 0.

exp = hyperbolic

Rewrite exponential functions in terms of hyperbolic functions.

gamma = binomial

Rewrite products of gamma functions into a binomial coefficient when possible. This converter looks for subexpressions of the form gamma(a+1)/(gamma(b+1)*gamma(a+1-b)) and replaces them with binomial(a,b). Only explicit occurrences of this three-factor pattern are transformed; the converter does not rewrite gamma(a) or gamma(b) individually, nor does it attempt to derive a binomial coefficient from expressions where the pattern is not present as an actual subexpression.

gamma = beta

Rewrite products of gamma functions into beta functions when possible. This converter looks for subexpressions of the form gamma(a)*gamma(b)/gamma(a+b) and replaces them with beta(a,b). Only explicit occurrences of this three-factor pattern are transformed; the converter does not rewrite gamma(a) or gamma(b) individually, nor does it attempt to derive beta(a,b) from expressions where the pattern is not present as an actual subexpression.

gamma_like = gamma

Normalize gamma-like functions to gamma.

hyperbolic = hyperbolic_tanh_half_angle

Rewrite hyperbolic functions in terms of the tanh half-angle substitution u = tanh(z/2), producing rational functions of u: 

sinh(z) = 2 u / (1 - u^2)
cosh(z) = (1 + u^2) / (1 - u^2)
tanh(z) = 2 u / (1 + u^2)
sech(z) = (1 - u^2) / (1 + u^2)
csch(z) = (1 - u^2) / (2 u)
coth(z) = (1 + u^2) / (2 u)
trig = trig_tan_half_angle

Rewrite trigonometric functions in terms of the tangent half-angle substitution t = tan(z/2), producing rational functions of t: 

sin(z) = 2 t / (1 + t^2)
cos(z) = (1 - t^2) / (1 + t^2)
tan(z) = 2 t / (1 - t^2)
sec(z) = (1 + t^2) / (1 - t^2)
csc(z) = (1 + t^2) / (2 t)
cot(z) = (1 - t^2) / (2 t)
exp = trig

Rewrite exponential functions in terms of trigonometric functions.

inverse_trig = log

Convert inverse trigonometric functions to logarithmic form.

hyperbolic = exp

Convert hyperbolic functions to exponential form.

trig = exp

Convert trigonometric functions to exponential form.

trig = sin_cos

Convert all six trigonometric functions to sin/cos form.

gamma = sin

Apply the identity gamma(x)*gamma(1-x) = %pi/sin(%pi*x).

trig = normalize_trig_argument

Normalize the argument of trigonometric functions when the argument is a first-degree polynomial in %pi.

hstep = signum

Convert hstep(x) into (1 + signum(x))/2.

signum = hstep

Convert signum(x) into 2*hstep(x) - 1.

abs = sqrt

Convert abs(x) into sqrt(x^2). When radexpand is true, this is simplified back to abs(x).

signum = abs

Convert subexpressions of the form X*signum(X) into abs(X). This converter does not rewrite signum(X) itself, since X*signum(X) is not a subexpression of signum(X).

abs = signum

Convert abs(x) into conjugate(x)*signum(x).

erf = hypergeometric

Convert erf(x) into (2*x/sqrt(%pi))*hypergeometric([1/2],[3/2],-x^2).

erf = integral

Convert erf(x) into an integral representation.

erfc = erf

Convert erfc(x) into 1 - erf(x).

erf = erfc

Convert erf(x) into 1 - erfc(x).

erf = erfi

Convert erf(x) into i*erfi(-i*x).

erfi = erf

Convert erfi(x) into -i*erf(i*x).

^ = trigreduce

Convert integer powers of trigonometric functions to a Fourier sum.

gamma_incomplete = expand

Expand incomplete gamma functions.

atan = log

Convert atan(x) to logarithmic form.

factorial = product

Convert n! to product(g,g,1,n).

binomial = factorial

Convert binomial(n,k) to factorial form.

log10 = log

Convert log10(x) into log(x)/log(10).

genfact = pochhammer

Convert genfact to pochhammer form.

tanh = sinh

Convert tanh(x) into sinh(x)/cosh(x).

tan = sin

Convert tan(x) into sin(x)/cos(x).

csc = sin

Convert csc(x) into 1/sin(x).

sin = sinc

Convert sin(x) into x*sinc(x).

sinc = gamma

Convert sinc(x) into 1/(gamma(1+x/%pi)*gamma(1-x/%pi)).

sinc = sin

Convert sinc(x) into sin(x)/x.
